Öffnet man eine parallele Region, so kann man festlegen, wie die Variablen innerhalb einer parallelen Region behandelt werden sollen.

shared (per default): Geteilt für alle Threads
Race-Conditions bei globaler Variable denkbar.

private: nicht initialisierte Kopie der Variable auf jedem Thread 
(Vorherig abgespeicherter Wert wird nicht übernommen für die Variable)

firstprivate: Jeder Thread bekommt eine Kopie der Shared Variable (Wert wird somit übernommen)
Diese Variable ist für jeden Thread lokal.
Am globalen Variablenwert wird sich hierdurch nichts verändern.
