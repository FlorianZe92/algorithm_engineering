simdlen: 
Breite von verwendeten Vektoren wird hierdurch vorgeschlagen (Vektorgröße) – Präferenz wird ihm mitgeteilt
Sollte man dem Compiler überlassen (Inkorrekte Angaben sorgen für Performance Verlust – Ergebnisse bleiben aber korrekt)
Meiste Compiler ignorieren simdlen (Intel Compiler achtet hierauf)


safelen: 
Vektoren von gewisser Größe dürfen nicht überschritten werden 
(Wenn man z.B. die Vektorgröße vorher begrenzt hatte und sonst der Code nicht korrekt ausführbar wird)
Maximale Vektorbreite kann sinnvoll sein bei vorhergehenden Abhängigkeiten.

aligned:
Compiler wird mitgeteilt, dass die angegebenen Vektoren innerhalb der for-Schleife aligned sind zu einer Byte Boundary 
(meist 64 Byte, da dies einer Cache line entspricht)
Pointer werden hierbei übergeben. (Vorher sicherstellen, dass die Daten auch wirklich aligned sind)
